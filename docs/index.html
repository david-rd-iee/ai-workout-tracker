Project Description

The AI Workout Tracker is a comprehensive cross-platform application designed to streamline the management of fitness training sessions between personal trainers and their clients. The application features real-time communication capabilities, allowing trainers and clients to interact seamlessly through instant messaging and group chats. Clients can easily book training sessions based on trainer availability, with integrated calendar support to manage schedules effectively.

Since the website was last refreshed on February 24, 2026, the codebase has expanded with structured workout logging for strength, cardio, and other activities; trainer-facing workout summaries with notes; workout history browsing and CSV export; Apple Health heart-rate tracking on iOS; and a regression-based exercise estimator pipeline that learns from logged workout data.

actor User
participant ChatPage as WorkoutChatbotPage
participant WorkflowSvc as WorkoutWorkflowService
participant EstimatorPrep as WorkoutWorkflowEstimatorPreparationService
participant SummaryProjection as WorkoutWorkflowSummaryProjectionService
participant ChatSvc as WorkoutChatService
participant ChatFn as workoutChatCallable
participant OpenAI
participant LogSvc as WorkoutLogService
participant CompleteFn as completeWorkoutEvent
participant FS as Firestore
participant PostWrite as onWorkoutEventCreated
participant RTDB as Realtime Database
participant HistoryPage as WorkoutHistoryPage
participant SummarySvc as WorkoutSummaryService
participant SummaryPage as WorkoutSummaryPage
participant ChatDetail as ChatDetailPage

User->>ChatPage: Enter workout details
ChatPage->>WorkflowSvc: processWorkoutMessage(message, messages, screenState)
WorkflowSvc->>EstimatorPrep: prepareEstimatorsForSession(session)
EstimatorPrep-->>WorkflowSvc: estimatorIds
WorkflowSvc->>ChatSvc: sendMessage(message, session, history, estimatorIds)
ChatSvc->>ChatFn: httpsCallable(payload)
ChatFn->>OpenAI: parse/update structured workout data
OpenAI-->>ChatFn: assistantMessage + summary
ChatFn-->>ChatSvc: ChatResponse
WorkflowSvc->>SummaryProjection: projectWorkflowState(updatedSession)
WorkflowSvc-->>ChatPage: session + live summary rows

User->>ChatPage: Submit workout
ChatPage->>WorkflowSvc: submitWorkout(session, requestTrainerNotes)
WorkflowSvc->>LogSvc: saveCompletedWorkout(session)
LogSvc->>CompleteFn: completeWorkoutEvent(normalized WorkoutEvent)
CompleteFn->>FS: Write users/{uid}/workoutEvents/{eventId}
CompleteFn-->>LogSvc: { eventId, status: persisted }

FS-->>PostWrite: Trigger on users/{uid}/workoutEvents/{eventId}
PostWrite->>FS: Update user stats + score derivations
PostWrite->>FS: Read same-day workoutEvents
PostWrite->>FS: Upsert users/{uid}/workoutSummaries/{YYYY-MM-DD}

alt user has trainerId
  PostWrite->>FS: Read users/{uid} / clients/{uid} for trainerId and user name
  PostWrite->>RTDB: Find or create direct chat with trainer
  alt first workoutSummary message for the day does not exist
    PostWrite->>RTDB: Create chats/{chatId}/messages/workout_summary_{YYYY-MM-DD}
    PostWrite->>RTDB: Set lastMessage + lastMessageTime
  else same-day message already exists
    PostWrite->>RTDB: Update existing workout_summary_{YYYY-MM-DD} message text
    PostWrite->>RTDB: Update lastMessage only
  end
end

LogSvc->>FS: Poll derivations/workout_summary + derivations/score_aggregation
LogSvc-->>WorkflowSvc: savedSession + scoreUpdate
WorkflowSvc->>SummaryProjection: projectWorkflowState(savedSession)
WorkflowSvc-->>ChatPage: saved state

User->>HistoryPage: Open workout history
HistoryPage->>SummarySvc: listRecentWorkoutSummaries(uid)
SummarySvc->>FS: Read users/{uid}/workoutSummaries order by date desc
SummarySvc-->>HistoryPage: grouped daily history cards

User->>SummaryPage: Open a specific day
SummaryPage->>SummarySvc: getWorkoutSummary(uid, date)
SummarySvc->>FS: Read users/{uid}/workoutSummaries/{date}
SummarySvc-->>SummaryPage: daily summary for display

User->>ChatDetail: Open trainer chat
ChatDetail->>RTDB: Subscribe to chats/{chatId}/messages
RTDB-->>ChatDetail: workout_summary message
ChatDetail-->>User: Render workout summary as custom card, not a normal bubble

Note over PostWrite,SummarySvc: The daily WorkoutSummary is the source of truth for history and persisted summary views.
Note over RTDB,ChatDetail: Trainer chat keeps one workout_summary message per day and updates it in place for additional same-day events.

Key Features

  • Real-time Communication: Instant messaging between trainers and clients with group chat support
  • Session Management: Booking system with availability scheduling, calendar integration, rescheduling, and cancellation support
  • AI Workout Logging: Chatbot-assisted logging for strength, cardio, and other workouts with normalized units and structured exercise data
  • Workout History & Export: Recent sessions can be reviewed in-app, grouped by date, and exported as CSV tables
  • Trainer Collaboration: Trainers can access client workout history and receive workout summaries with trainer notes
  • Health Tracking: Apple Health heart-rate sync, live-session monitoring, and workout insight graphs
  • Performance Scoring: Exercise estimators, expected-effort scoring, and automatic regression retraining for new workout samples
  • Leaderboards & Progression: Competitive rankings and Greek statue progression keep users motivated over time
  • Cross-Platform: Web and iOS delivery through Angular, Ionic, Firebase, and Capacitor
